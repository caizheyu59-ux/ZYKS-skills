**步骤 ①-2：获取详细搜索数据**

- **端点**：`http://apis.5118.com/keywordparam/v2`

- **参数**：`keywords={长尾词1|长尾词2|...}`（用"|"分隔）
- **返回**：每个长尾词的详细数据（两步调用：提交→获取）

**步骤 ①-3：获取竞争度数据（替代方案）**

- **当前方案**：使用 5118 的 `bidword_company_count`（竞价公司数）作为竞争度指标
- **原因**：Serper.dev API 暂时不可用（网络连接失败）
- **替代方案已启用**：配置文件中已标记 `kgr_alternative.enabled = true`

**步骤 ①-4：综合评分与筛选**

计算每个长尾词的综合得分（满分100）：

```
搜索量得分（40分）：
  - 流量指数：min(index / 500 × 20, 20)
  - 移动指数：min(mobile_index / 500 × 10, 10)
  - 抖音指数：min(douyin_index / 100 × 10, 10)

竞争度得分（30分）：
  - bidword_kwc：1=0分（高竞争）, 2=15分（中）, 3=30分（低）

KGR 评级加分（0~20分）：
  - KGR=0：+20分（蓝海冠军）
  - KGR<0.01：+15分
  - KGR 0.01-0.05：+10分
  - KGR>0.05：+0分

商业价值加分（0~15分）：
  - SEM价格上限 × 3，最高15分

扩展潜力加分（0~5分）：
  - long_keyword_count / 100000 × 5，最高5分

总分 = 搜索量得分 + 竞争度得分 + KGR评级 + 商业价值 + 扩展潜力
```

**筛选标准**：选择"转化高、竞争力中等"的长尾词
- bidword_kwc = 2（中等竞争）
- SEM价格 > 2（有商业价值）
- 优先选择"软件"、"工具"、"平台"等采购意图强的词

#### Kingsway 核心产品功能

- ✅ 使用真实的对比：vs YouTube、vs Vimeo、vs 自建服务器

**核心特性**：
- **Type × Style 体系**：6 种类型 × 8 种风格的自由组合
  - Type：infographic, scene, flowchart, comparison, framework, timeline
  - Style：notion, elegant, warm, minimal, blueprint, watercolor, editorial, scientific
- **智能分析**：自动识别文章类型和核心论点
- **自动适配**：根据 Kingsway 文章风格（A-G）自动选择配图方案
- **文字策略**：为避免中文生成错误，采用"少文字、多图标"策略

**配图规则**：
- 每篇文章 **3 张配图**
- 封面 **不需要配图**
- 每张图最多 **10 个中文字符**
- 优先使用 **图标和数字** 表达

**GeekAI API 配置**：
- 模型：nano-banana-2
- 端点：https://geekai.co/api/v1/images/generations
- 支持尺寸：1K, 2K, 4K
- 建议：使用 **1K**（速度快，成功率高）
